Week 02 — Early Sketching + UX Thinking

Date range: Jan 27 – Feb 2

1) Summary

  • ✅ Created initial sketches of device form
  • ✅ Planned dual-screen concept (OLED + E-ink)
  • ✅ Designed user flow: type → save → review → export
  • ✅ Started command-based interaction system

2) What I did

Design

  • Initial sketches — Created device form factor sketches inspired by DS-style compact devices.
  • Dual-screen concept — Planned OLED for typing feedback and e-ink for reading mode.
  • UX flow — Mapped user journey: type → save → review → export workflow.

Planning

  • Command system — Began designing command-based interaction (similar to Vi/Vim) for minimal UI.

3) Decisions made

  • Decision: Use dual-display approach (OLED + E-ink).
    • Reason: OLED provides instant feedback for typing, e-ink reduces eye strain during review.
    • Tradeoff: Increases complexity and power consumption.
  • Decision: Command-based interface instead of traditional GUI.
    • Reason: Minimal, distraction-free, and encourages intentional interaction.
    • Tradeoff: Steeper learning curve for new users.
